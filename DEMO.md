# مدیریت وظایف شخصی — دمو

نسخه دمو SPA فارسی (RTL) برای جلسه TPO.

## اجرا

```bash
npm install
npm run dev
```

مرورگر: `http://localhost:5173`

## قابلیت‌های دمو

- CRUD وظایف با ذخیره در `localStorage`
- وضعیت (انتظار / در حال انجام / انجام‌شده)
- Priority و Severity جداگانه
- Due Date و نمایش Overdue
- پروژه‌ها و برچسب‌ها
- جستجوی متنی و Regex
- فیلتر ترکیبی
- AI mock: پیشنهاد عنوان، تکمیل توضیحات، تقسیم Subtask
- ۴ وظیفه نمونه از پیش بارگذاری شده

## دمو سریع

1. وظیفه «تماس با بیمه» → Priority کم + Severity Critical + Overdue
2. جستجو: `JWT` یا Regex `^پیاده`
3. فیلتر: پروژه TPO + وضعیت انتظار
4. وظیفه جدید → توضیح بنویسید → ✨ پیشنهاد عنوان
5. «پیاده‌سازی JWT» → ✨ تقسیم به Subtask

## بازنشانی

دکمه «بازنشانی دمو» در سایدبار داده‌های نمونه را دوباره بارگذاری می‌کند.

## باز کردن مستقیم در مرورگر (دابل‌کلیک)

### چرا `dist/index.html` کار نمی‌کند؟

نسخه معمولی (`npm run build`) جاوااسکریپت را در فایل‌های جدا (`assets/*.js`) می‌گذارد. مرورگرها به‌دلیل امنیت، بارگذاری ES Module از پروتکل `file://` را **مسدود** می‌کنند — برای همین صفحه سفید می‌ماند.

Cursor ممکن است با سرور داخلی باز کند؛ مرورگر با دابل‌کلیک نه.

### راه‌حل: نسخه تک‌فایلی

```bash
npm run build:standalone
```

فایل **`dist-standalone/index.html`** ساخته می‌شود (~۱۶۰KB). همه JS و CSS داخل همان فایل است.

→ این فایل را **دابل‌کلیک** کنید یا برای دیگران بفرستید. فقط مرورگر لازم است (اینترنت فقط برای فونت فارسی).

## نمایش به دیگران (بدون نصب برای بیننده)

### روش ۱ — یک فایل HTML (ساده‌ترین)

```bash
npm run build:standalone
```

فایل `dist-standalone/index.html` را ایمیل / Telegram / USB بفرستید.

### روش ۲ — سرور محلی (برای ارائه)

```bash
npm run build
npx serve dist
```

آدرس `http://localhost:3000` را نشان دهید.

### روش ۳ — آپلود آنلاین

```bash
npm run build
npx surge dist
```

یا Cloudflare Pages / GitHub Pages / Vercel.

### خلاصه

| دستور | خروجی | دابل‌کلیک مرورگر |
|--------|--------|------------------|
| `npm run build` | پوشه `dist/` | ❌ |
| `npm run build:standalone` | `dist-standalone/index.html` | ✅ |
